Desenvolvimento Baseado em Componentes Definição

Foi criada na Conferência de Engenharia de Software da OTAN no ano de 1968. Tem como ideal "Crie uma vez, use onde quiser." Component-based Software Engineering (CBSE) é um processo que enfatiza o projeto e a construção de sistemas baseados em computador usando "componentes" de software reutilizáveis.

Características

Examinando essas definições, percebe-se que há um consenso quanto às características que devem estar presentes em um componente. Pode-se destacar:

Permitir o desenvolvimento de sistemas por montagem: um componente deve permitir que parte do (ou todo) desenvolvimento do sistema seja feito pela composição de partes existentes.

Possuir interfaces explícitas: o funcionamento do componente não pode depender do contexto onde estará presente, a não ser através de interfaces, sejam elas de exportação (serviços requeridos) ou de importação (serviços requeridos). Essa característica possibilita a diminuição do acoplamento do sistema.

Ser uma unidade auto-contida: isso significa que o componente deve funcionar independente de outro componente, sendo classificado assim como uma unidade de empacotamento e entrega.


Diferenças entre componentes e objetos

Componentes são independentes de linguagem. É possível a construção de um componente tanto em linguagens orientadas a objetos, quanto em linguagens não OO. Enquanto isso, objetos existem apenas em linguagens OO. Normalmente, a granularidade dos componentes é maior do que a dos objetos. Por serem auto-contidos, componentes são empacotados e entregues. Já os objetos necessitam de um contexto para funcionar, não podendo ser encarado como uma unidade de entrega. Em linguagens OO, a instanciação de um componente gera um objeto.
Componentes

O desenvolvimento de sistemas através de componentes consiste em criar o software utilizando componentes pré-construídos, não precisando assim partir do zero. Duas consequências diretas dessa definição são: a possibilidade de aumentar a produtividade da equipe e de reduzir os custos do projeto.

Entretanto, a utilização de componentes requer uma nova forma de pensar da equipe do projeto. Arquitetos, analistas e programadores devem preocupar-se agora com a integração do sistema com os componentes que serão utilizados, necessitando assim de uma atenção maior na definição de interfaces internas e externas. Esses pontos podem gerar mudanças no processo de desenvolvimento utilizado.


Seleção: Pesquisar e selecionar os possíveis componentes que poderão ser utilizados no desenvolvimento do sistema. Durante essa etapa, deve existir a preocupação em selecionar os componentes que atendam, total ou parcialmente, aos requisitos da arquitetura do sistema; além disso deve-se garantir que tais componentes são extensíveis e permitem adaptação.

Adaptação: Após selecionado, o componente deve ser adaptado às especificidades do sistema. Essa etapa é necessária porque dificilmente o componente se encaixará perfeitamente no sistema em desenvolvimento.

Composição: É o momento no qual o componente é conectado, através das interfaces, ao sistema. Trata-se da etapa principal do desenvolvimento de sistema através de componentes..
Vantagens e Desvantagens

Entre vantagens e desvantagens existentes na utilização de componentes para desenvolver software, é importante destacar a vantagem de que os componentes têm bom potencial para auxiliar o desenvolvimento de grandes sistemas. Por outro lado, uma desvantagem é que a curva de aprendizado pode ser desfavorável nos primeiros desenvolvimentos baseados em componentes, o que deve ser levado em consideração no planejamento desses projetos iniciais.
